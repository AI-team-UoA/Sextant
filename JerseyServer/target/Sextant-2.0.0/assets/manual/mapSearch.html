Search for maps in a registry

Sextant provides an alternative to load map using a search mechanism. Using this feature we can load maps without knowing their mapid and also find maps that meet certain criteria and may span over multiple map registries. We can access this mechanism by using button in Sextant's main menu bar and then select Search Endpoint as shown in the figure below.


We then have to fill the following form to search for maps. The first four fields in the form are used to provide keywords for the map metadata:

  • Title: Keywords to match in titles
  • Creator: Keywords to match with creators
  • License: Keywords to match licenses
  • Theme: Keywords to match themes

The next two fields are used to point the map registry where we will search for maps. In the URI field we provide the URI that is used to query the registry and in the Port field the port number in which the registry listens to. The user can leave the Endpoint Information fields empty. In this case Sextant will use the default map registry that is provided in the server-configuration.properties under /Sextant/WEB-INF/classes/server as described in Save map manual page.



The last element in the form is the Draw Extent button. When we click this button a mini map appears in the form where the user can draw a rectangle on this map. This rectangle is used to represent the area of interest and the results of our search will contain only maps that meet our keywords and spatialy intersect with the given area.
