Create a layer from a set of Predefined Queries

To assist non-exert users we provide an alternative to creating layers out of SPARQL endpoints using predefined queries. To create such a layer we use the button in Sextant's main menu bar and then select Predefined Queries as shown in the figure below.


We then have to fill in the following form to create our layer. First we need to provide the URI of an endpoint in the field URI and the port it listens to in the field Port:80 (by default the port is set to 80). We can then click the Connect button to communicate with the SPARQL endpoint and get a list of the available predefined queries in plain text as shown below. Then by simple selecting one of the queries we can create the corresponding layer.




These predefined queries are created by experts to assist simple users in visualizing the data in a SPARQL endpoint. Predefined queries follow the ontology shown in the image below.


Using this ontology, we can add predefined queries as data in a SPARQL endpoint. Below we have an example of a predefined query expressed in triple patterns:

<query/id/0> rdf:type <http://geo.linkedopendata.gr/query/ontology/predefinedQuery> .
<query/id/0> <http://geo.linkedopendata.gr/query/ontology/hasLabel> "plain text description" .
<query/id/0> <http://geo.linkedopendata.gr/query/ontology/isTemporal> "true/false" .
<query/id/0> <http://geo.linkedopendata.gr/query/ontology/hasText> "SPARQL query" .
