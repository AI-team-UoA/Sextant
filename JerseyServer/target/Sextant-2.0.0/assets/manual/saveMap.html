Creating and saving-updating a map

In Sextant we can group our layers under thematic maps. Each map we create is a resource that has a unique URI that can be used to share and load the map. To create or update a map we use the button in Sextant's main menu bar and then select Save map as shown in the figure below.


We then have to fill the following form to create or update our map. The first five fields in the form are the metadata that act as the map legend:

  • Title: A title for our map
  • Creator: The creator of the map
  • License: The license of the map
  • Theme: The map theme
  • Description: A small text that describes our map

The next four fields are used to select the map registry in which the map will be saved:

  • URI: The URI of the registry endpoint
  • Port: The port in which the endpoint listens to (default value is 80)
  • User: The username to connect to the endpoint
  • Password: The password to connect to the endpoint

To select if we want to create a new map or update the map that is currently loaded we use the Select create mode drop-down list. Finally we can choose which of the loaded layers will be part of our map by checking the checkbox in from of each layer's name.



Map registries are SPARQL endpoints that hold the information to recreate maps using Sextant. Each map is automatically transformed into a set of RDF triples that are used to represent it. The user can leave the Endpoint Information fields empty. In this case Sextant will use the default map registry that is provided in the server-configuration.properties under /Sextant/WEB-INF/classes/server. There are seven parameters that should be set to define our default registry in the configuration file:

  • HOSTNAME = URI of the registry host
  • PORT = The port the registry listens to
  • ENDPOINTNAME_STORE = The registry store URI
  • ENDPOINTNAME_QUERY = The registry query URI
  • ENDPOINTNAME_UPDATE = The registry update URI
  • USER_STORE = Username to connect to registry
  • PASS_STORE = Password to connect to registry

When we create/update the map two things change in the UI of Sextant. First, a new variable is added to the query string of the URL in our browser. This variable is named mapid and has as value the unique identifier for the map. This URL can be used to share and load the map.
