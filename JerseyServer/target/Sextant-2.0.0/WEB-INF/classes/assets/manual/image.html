Create a layer from an Image file

We can create a new layer from a geo-referenced image file using the button in Sextant's main menu bar and then select to Load GeoTIFF layer as shown in the figure below.


We then have to fill in the following form to create our layer. The first field is the name of the layer. Layer names are unique in Sextant, so each layer must have a different name. In the Image URI field we can provide a URI of a KML file in case it is a web resource, or we can click on the Browse button to select a local image file from our file system. In the GDAL info URI field we can provide a URI of a text file with the results of gdalinfo command in case it is a web resource, or we can click on the Browse button to select the text file from our file system.



The text file that contains the spatial metadata for the image must have the coordinated in EPSG:4326. To create this file we use the GDAL library and we run the following commands:

  • gdalwarp -t_srs EPSG:4326 image.tif image_4326.tif
  • gdalinfo image_4326.tif >> image_4326.txt

If we have an installation of the GDAL library in our system, Sextant can handle the creation of the text file automatically if we provide the path to the library in the server-configuration.properties file under /Sextant/WEB-INF/classes/server. In that case we can leaver the field GDAL info URI empty, and Sextant will automatically create the text file and extract the spatial information.
